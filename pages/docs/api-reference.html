Authentication

API Key Authentication

Include your API key in the header of each request:

Authorization: Bearer YOUR_API_KEY

Users API

GET /api/v2/users/{id}

Retrieve a user by their ID.

Parameters

Name Type Description
idRequired string The unique identifier of the user
fieldsOptional string[] Comma-separated list of fields to include

Response

{
  "id": "usr_123",
  "name": "John Doe",
  "email": "john@example.com",
  "created_at": "2025-08-25T00:00:00Z",
  "role": "admin"
}

POST /api/v2/users

Create a new user.

Request Body

{
  "name": "string",
  "email": "string",
  "role": "string",
  "settings": {
    "theme": "light|dark",
    "notifications": boolean
  }
}

Response

{
  "id": "usr_123",
  "name": "John Doe",
  "email": "john@example.com",
  "created_at": "2025-08-25T00:00:00Z",
  "role": "admin"
}

Rate Limits

Rate Limiting

Our API implements rate limiting to ensure fair usage:

  • Free tier: 100 requests per minute
  • Pro tier: 1000 requests per minute
  • Enterprise tier: Custom limits

Error Codes

Code Description
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid API key
403 Forbidden - Insufficient permissions
404 Not Found - Resource doesn't exist
429 Too Many Requests - Rate limit exceeded
